# ModelWiki 产品愿景与边界

## 1. 产品定位

ModelWiki 是一个面向手办、模型、拼装模型、景品和收藏类周边爱好者的网站。

它同时承担四类职责：

1. **资料库**
   - 查询商品资料
   - 查看图片、规格、厂商、系列、角色、发售和价格信息

2. **个人收藏管理**
   - 已拥有
   - 想要
   - 已预订
   - 已出售
   - 关注
   - 个人备注、价格、渠道、日期、数量、隐私设置

3. **轻量社区**
   - 点赞
   - 收藏
   - 评论
   - 评分（可选）
   - 热门榜单
   - 最近评论
   - 公开收藏页（可选）

4. **内容与评测平台**
   - 管理员/编辑者发布手办评测
   - 购买体验
   - 开箱
   - 做工、涂装、可动性、性价比分析
   - 系列盘点
   - 对比评测
   - 再版与购买建议

## 2. Crawler 的定位

Crawler 不是长期产品中心。

它主要用于：

- 网站冷启动；
- 批量建立基础资料库；
- 补充缺失字段；
- 少量来源更新；
- 管理员主动触发的 refetch；
- 发现明显数据风险并提交人工审核。

网站进入稳定期后，主线应转为：

> 人工维护 + 用户收藏 + 社区互动 + 编辑评测 + 小规模自动补全

禁止把 ModelWiki 重新设计成“不断扩大爬虫规模”的项目。

## 3. 已确认的核心业务决策

### A. 人工决定优先于自动规则

已确认：**是**。

机器负责发现风险，人工负责最终决定。

自动检测不能无条件覆盖人工已经确认的结果。

### B. 图片数量少，但人工确认质量足够，允许通过

已确认：**允许**。

例如：

- 当前只有 1–2 张图片；
- 但人工确认有一张可靠、清晰、相关的商品图；

则可以 resolved。

因此机器检测条件和人工接受标准不要求机械对称。

### C. 人工已处理且证据没变化，禁止重复生成相同审核项

已确认：**禁止重复生成**。

必须通过：

- decision record
- evidence fingerprint
- suppression/reopen rule

保证人工决定具有稳定性。

### D. 图片来源采用分级策略

已确认：

- 可信官方/零售来源可以自动进入正式图库；
- MFC 用户上传、来源不确定、共享候选、疑似房间/展示柜/合集图等高风险图片必须先进入 candidate review；
- 不确定来源进入 manual。

### E. 详情数据采用分级策略

已确认：

- 可信结构化来源可以自动写入；
- 冲突、稀疏、异常值进入 detail_review；
- 人工确认后发布；
- 自动系统不得覆盖人工确认的字段，除非有明确的版本/变更策略。

## 4. 产品成功标准

ModelWiki 的成功不是：

- 抓了多少页面；
- 创建了多少 crawler job；
- 队列有多少条；
- 数据库有多少条记录。

而是：

- 用户能快速找到可信资料；
- 图片相关、清晰、可放大；
- 收藏管理有长期使用价值；
- 评论和社区信号真实；
- 管理员能稳定审核；
- 人工决定不会被机器反复覆盖；
- 编辑者可以持续发布评测和高质量原创内容；
- 自动化降低维护成本，而不是制造更多脏数据。
