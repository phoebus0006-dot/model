## 2. Reconciliation Execution Plan

### Phase 1: Account System Fix (parallel Agents A + B)

**Agent A (frontend-email-auth):**
1. Create migration to ADD email columns to users table:
   - `email TEXT UNIQUE NOT NULL`
   - `normalized_email TEXT UNIQUE NOT NULL`
   - `email_verified_at TIMESTAMP`
   - `email_verify_token_hash TEXT`
   - `email_verify_expires_at TIMESTAMP`
2. Update schema.prisma User model (email required, normalizedEmail, emailVerifiedAt, etc.)
3. Rewrite auth.ts:
   - registerSchema: email + password + displayName
   - loginSchema: email + password only
   - Remove `app.prisma as any`
   - Implement email normalization (trim + lowercase domain)
   - Implement email verification with hashed tokens
   - Implement password reset with anti-enumeration
4. Write real DB integration tests

**Agent B (guanli-admin-auth):**
1. Create migration for `admin_accounts` table + `admin_audit_logs` table
2. Create AdminAccount model in schema.prisma
3. Create `/admin/auth/login`, `/admin/auth/logout` routes
4. Create admin auth middleware (separate JWT audience, separate cookie)
5. Create AdminAccount initialization CLI
6. Write admin auth tests

**Dependency:** Agents A and B can work in parallel (different files/models).
